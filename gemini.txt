### Flaw 1: The Broken English Morphology Trap (Stage 1 - FATAL)

**The Issue:**
In Fix 16 (Stage 1), you correctly identified that you need to expand verbs to catch past tense and progressive forms. However, the string-manipulation logic used to expand the verbs is overly simplistic:
```python
if verb.endswith("e"):
    # add 's', 'd', 'ing'
else:
    expanded_verbs.add(verb + "s")
    expanded_verbs.add(verb + "ed")
    expanded_verbs.add(verb + "ing")
```
**The Impact:**
English morphological rules are complex. Look at the verbs in your `ALGORITHMIC_VERBS` list:
* **"classify"** becomes `"classifys"` and `"classifyed"` (instead of *classifies, classified*).
* **"match"** becomes `"matchs"` (instead of *matches*).
* **"embed"** becomes `"embeded"` (instead of *embedded*).
* **"infer"** becomes `"infered"` (instead of *inferred*).

Because `TfidfVectorizer` looks for *exact* string matches, highly algorithmic papers using the words *"classified," "embedded,"* or *"matches"* will score exactly 0 and be silently thrown out.

**The Fix:**
Do not write manual string conjugation rules. Instead, use a proper stemming algorithm within the `TfidfVectorizer` to map all surface words back to their root algorithm automatically. `nltk`'s SnowballStemmer is lightweight, lightning-fast, and built exactly for this.

* **Update `src/stage1_tfidf_filter.py` (Replace Step 1 and Fix 16 logic):**
```python
    import nltk
    from nltk.stem.snowball import SnowballStemmer
    
    # ── FIX 16 (REVISED): Proper Algorithmic Stemming ───────────────────
    # Instead of manual string manipulation, we use a custom tokenizer 
    # that stems the abstract text so it perfectly matches our base verbs.
    stemmer = SnowballStemmer("english")
    
    # Stem our base verbs to ensure perfect alignment
    stemmed_vocab = list(set([stemmer.stem(v) for v in ALGORITHMIC_VERBS]))
    log.info(f"Stemmed {len(ALGORITHMIC_VERBS)} base verbs into {len(stemmed_vocab)} root tokens.")

    def stem_tokenizer(text):
        # Extract alphabetic words and stem them
        import re
        words = re.findall(r"(?u)\b[a-zA-Z][a-zA-Z]+\b", text.lower())
        return [stemmer.stem(w) for w in words]

    # ── Step 1: Build TF-IDF vectorizer ──────
    vectorizer = TfidfVectorizer(
        vocabulary   = stemmed_vocab,
        tokenizer    = stem_tokenizer,
        lowercase    = False, # handled in tokenizer
        ngram_range  = (1, 1)
    )
```

### Flaw 3: Arbitrary Character Truncation Breaks spaCy (Stage 4 - MINOR)

**The Issue:**
In `src/utils/graph_utils.py`, you extract and parse the method texts using hard character slicing:
`method_text = "\n".join(lines[method_start:method_end])[:5000]`
And again in the parser:
`doc = nlp(text[:4000])`

**The Impact:**
Slicing a string exactly at the 4,000th character will almost certainly slice a word in half (e.g., "...we applied an `optim`"). When spaCy's dependency parser hits the broken token `"optim"`, it will fail to categorize it as a verb, silently destroying the dependency tree for the final, most crucial sentence of the extracted methodology.

**The Fix:**
Slice by *words* (tokens) rather than raw characters. This guarantees you never feed a bisected word into the NLP engine.
